00 SQL注入导论

ZeroP零点 2021-09-16 10:32:07
Categories： Tags：

注入背景

原理

Web应用程序对用户输入的数据校验处理不严或者根本没有校验，致使用户可以拼接执行SQL命令。可能导致数据泄露或数据破坏，缺乏可审计性，甚至导致完全接管主机。

注入类型

布尔盲注：根据返回页面类型条件判断真假

时间盲注：用页面返回时间是否增加判断是否存在时间型盲注

错误注入：页面会返回错误信息（一般能报出物理路径）

联合查询注入：可以使用union的情况下可以使用联合注入

堆查询注入：可以同时执行多条语句

防御方法

使用参数化查询

目前市面上主流的框架基本都会有预处理SQL语句，都是在数据库完成 SQL指令的编译后才会套用参数，避免了参数中的恶意代码在数据库中执行
